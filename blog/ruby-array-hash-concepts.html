Ruby: Arrays and borders

January 18, 2016

Today, we will discuss about Ruby, more precisely Arrays and hashes.

What is the difference between an array and a hash

Array Hash
An ordered collection of objects An ordered collection of objects
Stores objects, arrays, hashes, classes, true and false Stores objects in pair (key, value)
Order of objects remain Order as how the keys were inserted
Index by 0,1,2,3,4,5... Index per key

We could say that an array is similar to a hash where the keys are imply and consecutive integers starting with zero. While the hash is similar to an array where the indexes can be anything.

When and why would we use an array? To keep a list. It could be a list of numbers, a list of strings, or a list of both string and integers or even a list of arrays. For example:

  • [1,2,3,4,5]
  • [Canada, United-States, France, England]
  • [[China,Japan,South Korea],[Uganda,Kenya, Tanzania]]

When and why would we use a hash? To keep a list with values. For example:

  • A list of States with their abbreviation
    • "California", "CA"
    • "Washington","WA"
  • A list of birthay
    • "John Doe","01-01-1901"
    • "Janet Die","12-12-1912"

Hash would therefore be really good to search a specific value: When is John Doe's birthay? What is the abbreviation for Dakota?

There are several ways to create an array or a hash. Here is my favorite for each of them:

  • array=[Canada, United-States, France, England] I have created a new array with four strings.
  • state_hash={"California" => "CA", "Washington" => "WA"}I have created a new hash table.

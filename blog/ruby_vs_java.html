Ruby vs JavaScript

February 12, 2016

My topic today is about Ruby and JavaScript. I will raise a few differences I have noticed between these two programming languages.

What I realized me at first is the need to declare a variable in JavaScript. We do not need to do this in Ruby, most likely because Ruby is a more recent language.

I noticed the need to use semi-colon at the end of a line for JavaScript while these are foregone in Ruby.

In Ruby, we talk about a hash while in JavaScript, we talk about an object. The terminology of their parts is a bit different but they basically work the same way. For example, a key in Ruby is called a property in JavaScript but their value is called value in both languages.

The one thing that really caught my attention is the for loop in JavaScript. How do I love the structure! It is built on one line in three parts. The first part initializes the loop. The second part gives the point where the loop should stop, and the last part updates the state. It is written this way:

for (var triangle = "#"; triangle.length <= #; triangle += "#")

Up until now, reading about JavaScript has been more difficult than reading about Ruby. I am hoping though, that we can build on our knowledge of the Ruby language to be successful in writing code in JavaScript.

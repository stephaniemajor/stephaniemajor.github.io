Ruby: Enumerable Method, Map

January 29, 2016

My topic today is about Ruby. More precisely about the enumerable method map.

What is an enumerable method?

An enumerable method is a “mix in” method (using another method) with the ability to sort and rely on an ordering between members of a collection. A collection is, more or less, a list of items (range, arrays, hashes, sets).

The enumerable method is built over the each method. Because it is built over an each method, only classes where the each method can be used may use an enumerable methods. For example, the array class and the hash class are good candidates because they produce or are a collection, therefore, perfect for an each method.

What does the enumerable method map do?

To start, maybe I should mention why I chose to talk about the map method: Because I have seen it used a lot and I wasn’t very clear about what it does. When I looked at it, it was as if, it was too simple to be true. So, I decided to investigate….

map creates an array with the results from running a block of code for every element in the enumerator. So yes, it is really that simple!

For example:

(1..4).map { |i| i*i} #=> [1,4,9,16]

But what does it do if no “block of code” is given?

(1..4).map { “cat”} #=> [ “cat”, “cat”, “cat”, “cat” ]

Voila! No trick, nothing hidden… except, maybe….. I should mention that instead of using the map method, we can use the collect method. They do exactly the same thing. I wouldn’t be surprised that these two synonymous methods come from how people visualize the process: some visualize it as “doing something” and “collecting” the results, while others see it as “re-mapping” the original object through some sort of transformation.

That’s it for today!
